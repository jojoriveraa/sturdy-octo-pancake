org.springframework.data.util

Class NullableUtils

  • java.lang.Object
    • org.springframework.data.util.NullableUtils

  • public final class NullableUtils
extends Object
    Utility methods to introspect nullability rules declared in packages, classes and methods.

    Nullability rules are declared using NonNullApi and Nullable and JSR-305 Nonnull annotations. By default (no annotation use), a package and its types are considered allowing null values in return values and method parameters. Nullability rules are expressed by annotating a package with a JSR-305 meta annotation such as Spring's NonNullApi. All types of the package inherit the package rule. Subpackages do not inherit nullability rules and must be annotated themself. 

     @org.springframework.lang.NonNullApi
 package com.example;
    Nullable selectively permits null values for method return values or method parameters by annotating the method respectively the parameters: 
     public class ExampleClass {

 String shouldNotReturnNull(@Nullable String acceptsNull, String doesNotAcceptNull) {
 // …
 }

 @Nullable
 String nullableReturn(String parameter) {
 // …
 }
 }

    Nonnull is suitable for composition of meta-annotations and expresses via Nonnull.when() in which cases non-nullability is applicable.
